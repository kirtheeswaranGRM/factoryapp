# FACTORY MANAGEMENT SYSTEM - FULL APPLICATION WORKFLOW

This document explains the full operational flow of the Factory Management System, including the interactions between the Mobile Frontend, Node.js Backend, and PostgreSQL Database.

## 2. DETAILED FLOW EXPLANATION

### A. Authentication & Authorization
- **Role Selection**: Users select their department (Operator, Production Head, Quality) and log in with email/password.
- **Attendance Trigger**: When a Production Head or QA staff logs in, the system automatically logs their login time into the `head_attendance` table via the `/head-attendance/log` API.

### B. Operator Workflow (Production Execution)
1. **Machine Selection**: Operators see a live list of all machines (`UNIT 1`, `UNIT 2`, etc.) and their current status (Running, Paused, Idle).
2. **Setup**: Before starting a machine, operators must select the **Product**, **Materials** (up to 4 types), **Color**, **Mold**, and **Cavity**.
3. **Execution**: 
   - Starting the machine sets a `start_time` in the DB.
   - The frontend calculates "Live Pieces" using the formula: `Floor(Elapsed Seconds / Cycle Time) * Cavity`.
4. **Hourly Reporting**: Every hour (or manually), operators log checks for Chiller, Compressor, Mold, and Machine status along with hourly output.
5. **Final Submission**: When production stops, a final report is generated and sent for approval. The machine status returns to 'IDLE'.

### C. Production Head Workflow (Management & Oversight)
1. **Approvals**: The PHead reviews pending production logs submitted by operators. They can Approve or Reject reports. Approved reports are used for final stock/reporting.
2. **Monitoring**: A real-time view of the entire factory floor status, showing which machines are running and their current output counts.
3. **Inventory**: Management of Raw Materials, Colors, Molds, and Finished Products. Includes updating stock levels and adding new items.
4. **Attendance**: Marking and reviewing attendance for the general staff and operators.

### D. Quality Assurance Workflow (Inspection)
1. **Periodic Checks**: QA staff move from machine to machine.
2. **Inspection**: For each machine, they record the average weight of parts, rejection pieces, and general remarks.
3. **Logging**: Inspections are saved via the `/qc_logs` API for quality trend analysis.

### E. Master Controls
- **Power Toggle**: In case of a power cut or factory-wide pause, the Operator screen provides "PAUSE ALL" and "RESUME ALL" buttons.
- **Logic**: Pausing calculates the current production and freezes the `accumulated_output` so no data is lost during the downtime.

## 3. DATA ARCHITECTURE (Core Tables)
- `users`: Authentication and roles.
- `machine_status`: Real-time state of each machine.
- `production_logs`: History of production runs (Pending/Approved).
- `inventory_*`: Stock levels for materials, products, and molds.
- `hourly_logs`: Granular production data points.
- `qc_logs`: Quality inspection records.
- `attendance`: Staff daily attendance.
